## Задача 1

### Текст задачи

На основе списка, содержащего двоичные значения цифр от 1 до 9, получить их десятичные представления. Все исходные данные вводятся с клавиатуры, организована проверка вводимых значений.

### Алгоритм решения

1)Создаем последовательность validBinaries, содержащую строковые представления двоичных чисел от 1 до 1001.

2)Функция getBinaryValue запрашивает ввод и с помощью Seq.contains и проверяет, входит ли введенная строка в эталонную последовательность. Если ошибка, то запрос повторяется для того же индекса.

3)В функции toDecimal строка разворачивается (Seq.rev), а затем через Seq.mapi каждый бит умножается на $2^{index}$.

4)С помощью Seq.map порядковые номера трансформируются в проверенные десятичные значения.

5)Результаты выводятся в виде списка кортежей (номер, значение) с помощью Seq.iter.

## Задача 2

### Текст задачи

Список содержит символы, обозначающие шестнадцатеричные цифры. Получить десятичное представление шестнадцатеричного числа из этих цифр с использованием Seq.fold.

### Алгоритм решения

1)Программа запрашивает количество символов и организует поочередный ввод через getValidHexChar.

2)Каждый символ проверяется на принадлежность к диапазону 0-9 или A-F.

3)Применяется Seq.fold.
    -Начальное значение 0L (int64).
    -На каждом шаге текущая сумма умножается на 16 и прибавляется значение нового символа.

4)Выводится исходная собранная строка и её итоговое десятичное представление.

## Задача 3

### Текст задачи

Получить для подкаталогов заданного каталога список кортежей вида
(название подкаталога, количество файлов в нем). Примечание.
Количество файлов указывается непосредственно для самого
подкаталога, исключая вложенные в него другие каталоги.

### Алгоритм решения

1)Программа запрашивает путь к директории и проверяет его существование через Directory.Exists.

2)Используется Directory.EnumerateDirectories для получения последовательности подпапок. Это позволяет обрабатывать данные без загрузки всего списка в память.

3)Каждый путь преобразуется в кортеж:
    -Первый элемент: Имя папки, извлеченное через Path.GetFileName.
    -Второй элемент: Количество файлов в этой папке, полученное через 
        Directory.EnumerateFiles и Seq.length.

4)Обработки UnauthorizedAccessException (ошибка доступа к системным папкам).

5)Полученная последовательность кортежей форматируется в таблицу и выводится на экран.
